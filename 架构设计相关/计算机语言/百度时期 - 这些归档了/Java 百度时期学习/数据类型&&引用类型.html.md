数据类型：

</div>

-   -   -   c、cpp中允许编译器建议寄存器的分配方式。java
            不行。寄存器位于cpu、最快。

        -   堆栈 

        -   -   位于通用寄存器（RAM区）
            -   某些对象的引用、具体对象不在这里
            -   对象的生存周期

        -   堆

        -   -   通用的内存池（RAM区）
            -   堆中没必要准确知道对象、数据存活多久
            -   缺点：内存分配、清理性能差

        -   常量

        -   -   编译到代码中的

        -   非RAM存储

        -   -   流对象、持久化对象
            -   在需要时转化为常规的、基于RAM的对象

        -   BigInteger（支持任何精度的整数）、BigDecimal（支持任何精度的定点数）

        -   定点数：小数点固定在最后一位称为定点整数、固定在最高位之后称为定点小数

        -   当创建一个数组对象时，相当于创建一个数组对象的引用、自动初始化为空

        -   实例化对象时，编译器初始化各个元素，元素：（基本类型：0、常规类的对象
            null）
